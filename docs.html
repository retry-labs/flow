Documentation

Flow reference.

Everything you need to write, style, and integrate diagrams with Flow. If something's missing, open an issue.

#Install

Flow ships as a single package that works in every major framework. See the landing page for per-framework snippets.

zsh
$ npm install flow-diagram

# or, zero-install via CDN
$ curl -O https://unpkg.com/flow-diagram/dist/flow-diagram.standalone.js

#Quickstart

Write a DSL string, pass it to parseDSL, then renderSVG. Done — no bundler, no peer deps, no canvas.

quickstart.js
import { parseDSL, renderSVG } from 'flow-diagram'

const graph = parseDSL(`
nodes:
  - id: client, kind: actor,   label: Client
  - id: api,    kind: service, label: API
  - id: db,     kind: store,   label: Postgres

edges:
  - client -> api, label: HTTPS
  - api    -> db,  label: query
`)

const svg = renderSVG(graph, { styleName: 'sleek' })
document.getElementById('arch').innerHTML = svg

#DSL grammar

Flow DSL is a YAML-ish format with four sections. All sections are optional except at least one of nodes:.

diagram.flow
# Optional meta directives at the top
style: sleek            # one of: sleek, sketch, iso, blueprint, city
title: "My architecture"

# Nodes — `id` is required. Everything else is optional.
nodes:
  - id: client,  kind: actor,   label: Client
  - id: api,     kind: service, label: API

# Edges — arrow shorthand: "->" solid, "..>" dashed
edges:
  - client -> api, label: "HTTPS"

# Optional playback steps for animation / narration
steps:
  - title: "Client request", nodes: [client, api], edges: [e-client-api-0]

Meta directives

KeyTypeDescription
stylestringRender style — sleek, sketch, iso, blueprint, or city.
titlestringOptional diagram title. Used by some styles and exported metadata.

Nodes

Every node needs an id. Coordinates (x, y, w, h) are optional — if any node omits them, auto-layout positions every non-boundary node left-to-right by rank.

KeyTypeDescription
idstringRequired. Used for edge endpoints and step references.
kindstringSemantic kind — see all 26 kinds. Defaults to service.
labelstringMain label shown in the node.
substringOptional sub-label (e.g. version, role).
shapestringOverride the default shape for this kind — rect, pill, diamond, hex, circle, cylinder, etc.
x, ynumberOptional absolute coordinates. Omit to trigger auto-layout.
w, hnumberNode dimensions. Defaults: w: 140, h: 70.
srcstringFor kind: image, an image URL to embed.

Edges

Edges use an arrow shorthand: -> (solid) or ..> (dashed). Extra keys follow after a comma.

edges.flow
edges:
  # Simple solid
  - client -> api
  # Solid with a label
  - api -> db, label: "query"
  # Dashed (async / cache read)
  - api ..> cache, label: "read"
  # Explicit kind for special variants
  - user -> login, kind: secure, label: "creds"
  - check -> deny, kind: error,  label: "no"
  - edge -> bus,   kind: realtime, label: "ws"

Auto-generated edge ids follow the pattern e-<from>-<to>-<index>. Use these in steps to light up specific edges.

Steps (playback)

Steps drive the optional active-node / active-edge highlighting. Pass step ids via activeNodes/activeEdges at render time, or iterate through graph.steps for timeline playback.

steps.flow
steps:
  - title: "Client request",  nodes: [client, gateway], edges: [e-client-gateway-0]
  - title: "Gateway fan-out", nodes: [gateway, orders],  edges: [e-gateway-orders-0]

#Node kinds

All 26 built-in kinds. Each maps to a semantic body rendered by the active style — use the right kind and the style layer handles the rest.

Need something off-menu? Use kind: service with an explicit shape: override for full control.

#Edge kinds

Edge kind drives stroke style, arrow marker, inline badge (secure / error), and active animation.

#Render styles

The same graph renders in five distinct styles. Set it globally with the style: directive or per-render with renderSVG(graph, { styleName }).

Tip: city uses a true isometric projection and depth-sorts nodes + edges. It's beautiful but slightly slower — prefer iso for realtime-updating dashboards.

#Zoom · Pan · Fullscreen · Download

Every mounted viewport ships with a floating control bar in the top-right corner. The same controls are available in every framework wrapper — they're built into the viewport, not bolted on at the page level. The controls only appear on hover so they don't compete with the content.

ControlMouseKeyboardAPI
Zoom in / outScroll-wheel over the diagram (with the floating + / − buttons as fallback).—vp.setZoom(z)
PanClick-and-drag anywhere on the diagram.—vp.setPan(x, y)
Reset viewFloating ⊙ button.—vp.resetView()
FullscreenFloating expand button (uses the native Fullscreen API).Esc to exitvp.toggleFullscreen()
Download SVGFloating ⬇ button — exports a self-contained SVG with fonts inlined.—vp.download()

Pass controls: false to mount() (or omit the corresponding wrapper props) to hide the floating panel — useful for embeds.

#Step player

If your graph declares a steps: section, the viewport adds a prev / play / step-counter / next group to the floating panel and renders a caption below the diagram. Active nodes and edges from the current step light up automatically — no extra wiring needed.

steps + autoplay
const vp = FlowDiagram.mount('#arch', {
  graph: FlowDiagram.parseDSL(dsl),
  styleName: 'sleek',
  autoplay: true,         // start playing immediately
  interval: 2200,         // ms between steps
  showPlayer: true,       // default; false hides the prev/play/next group
  onStepChange(i, step) {
    console.log('step', i, step.title);
  },
})
OptionTypeDescription
autoplaybooleanStart the timer immediately on mount.
intervalnumberMilliseconds between steps. Default 2000.
showPlayerbooleanShow the prev/play/next group. Default true when graph.steps exists.
stepIndexnumberInitial step index. Default 0.
onStepChangefnCalled with (index, step) on every step change.

#Click handlers

Every node is rendered with a data-node-id attribute, and edges with data-edge-id. The viewport wires these to onNodeClick / onEdgeClick callbacks — perfect for popovers, deep-links, or driving an external inspector.

click handlers
FlowDiagram.mount('#arch', {
  graph,
  onNodeClick(id) { openInspector(id); },
  onEdgeClick(id) { highlightEdge(id); },
});

The web-component dispatches matching CustomEvents — node-click, edge-click, and step-change — bubbling up to your framework's listener of choice.

#JavaScript API

Everything is tree-shakeable from the flow-diagram package.

renderSVG(graph, options) → string

Serializes a graph to a self-contained SVG string. Safe for SSR.

OptionTypeDescription
styleNamestringOne of sleek, sketch, iso, blueprint, city. Graph's style: directive wins if set.
activeNodesstring[]Node ids to highlight with active-state glow.
activeEdgesstring[]Edge ids to animate as "in flight".
paddingnumberExtra padding around the canvas (default 0).

mount(target, options) → viewport

Mounts an interactive viewport into a DOM element with built-in zoom, pan, fullscreen, and SVG download controls. Use this on plain HTML pages — the React / Vue / Angular wrappers ship the same controls automatically. The web-component <flow-diagram> uses mount internally.

vanilla.js
const vp = FlowDiagram.mount('#arch', {
  graph: FlowDiagram.parseDSL(dsl),
  styleName: 'sleek',
  // optional: activeNodes, activeEdges, padding,
  //           controls (default true), fileName,
  //           onNodeClick(id), onEdgeClick(id)
})

vp.setStyle('city')                       // swap render style
vp.setActive(['client'], ['e1'])          // highlight nodes/edges
vp.update({ graph: nextGraph })           // replace graph entirely
vp.setZoom(1.5); vp.resetView()           // programmatic zoom
vp.toggleFullscreen(); vp.download()      // controls API
vp.destroy()                              // clean up the DOM
OptionTypeDescription
graphobjectResolved graph object (from parseDSL or hand-written).
styleNamestringInitial render style. Falls back to graph.style, then 'sleek'.
activeNodesstring[]Initial active-node ids.
activeEdgesstring[]Initial active-edge ids.
controlsbooleanShow the floating control bar (default true).
fileNamestringStem used for the downloaded SVG (default flow-diagram).
onNodeClickfnCalled with the clicked node id (when nodes carry data-node-id).
onEdgeClickfnCalled with the clicked edge id.

The container element should have a fixed height (e.g. height: 420px) — the viewport fills it and the SVG is scaled with preserveAspectRatio.

Player methods on the viewport handle

When graph.steps exists, the handle returned by mount() also exposes:

MethodDescription
vp.play()Start (or resume) the autoplay timer.
vp.pause()Stop the autoplay timer (keeps current step).
vp.togglePlay()Toggle between play and pause.
vp.nextStep() / vp.prevStep()Advance / rewind one step (wraps).
vp.gotoStep(i)Jump to step i. Negative indices count from the end.
vp.setInterval(ms)Change the autoplay interval at runtime.
vp.stepIndex (getter)Current step index.

Same methods are also available as imperative properties on the <flow-diagram> web component (el.play(), el.nextStep(), etc.).

parseDSL(text) → graph

Parses the DSL text above into a graph object. Leaves x/y undefined if you omit them, letting resolveGraph auto-layout.

graphToDSL(graph, { compact }) → string

Inverse of parseDSL. Useful for "view source" tooling and round-trip tests. compact: true skips x/y/w/h so you get the clean version.

resolveGraph(graph) → graph

Low-level: runs auto-layout (if needed), assigns edge anchors, routes edges, and sorts nodes for depth. Called internally by renderSVG.

<FlowDiagram> — React

App.jsx
import { FlowDiagram } from 'flow-diagram'

<FlowDiagram
  dsl={dsl}                // or config={graphObject}
  style="sleek"
  activeNodes={['client','api']}
  activeEdges={['e-client-api-0']}
  animate={true}
  onNodeClick={n => console.log(n)}
  onEdgeClick={e => console.log(e)}
/>

Peer deps: react >= 17, react-dom >= 17.

<FlowDiagram> — Vue 3

App.vue
import { FlowDiagram } from 'flow-diagram/dist/wrappers/vue.js'

<FlowDiagram
  :dsl="dsl"                 // or :config="graph"
  style="city"
  :active-nodes="['client']"
  :active-edges="[]"
  @node-click="onNode"
  @edge-click="onEdge"
/>

Peer dep: vue >= 3.

<flow-diagram> — Web Component (Angular-friendly)

component.html
<!-- Attributes: dsl, config, diagram-style, active-nodes, active-edges,
     height, width, autoplay, interval, step-index, show-player, controls -->
<flow-diagram
  [attr.dsl]="dsl"
  diagram-style="blueprint"
  active-nodes="client,api"
  autoplay="true"
  interval="2200"
  height="420px"
  (node-click)="onNode($event)"
  (step-change)="onStep($event)">
</flow-diagram>

Register once by importing 'flow-diagram/dist/wrappers/angular.js' (or just the standalone bundle — it auto-registers). The element dispatches node-click, edge-click, and step-change CustomEvents and also exposes imperative .play() / .pause() / .nextStep() / .gotoStep() / .setStyle() / .download() methods.
