Why Flow

Small, sharp, and framework-agnostic.

Designed for docs sites, README hero images, Storybook stories, runbooks, and live-updating system dashboards.

5 built-in styles

Switch between sleek, hand-drawn sketch, isometric, CAD blueprint, and a full 3-D isometric "city" with one prop.

Smart auto-layout

Skip the x/y coordinates — Flow lays nodes out left-to-right by longest path. Override only when you want.

Playback steps

Narrate flows step-by-step. Built-in active-node glow and edge animations make runbooks feel live.

Runs anywhere

React component, Vue component, Angular web-component, or a plain renderSVG() call. SSR-safe.

1. Install

zsh
$ npm install flow-diagram

2. Use the component

App.jsx
// Peer deps: react >= 17, react-dom >= 17
import { FlowDiagram } from 'flow-diagram'

const dsl = `
nodes:
  - id: client, kind: actor,   label: Client
  - id: api,    kind: service, label: API
  - id: db,     kind: store,   label: Postgres

edges:
  - client -> api, label: HTTPS
  - api    -> db,  label: query
`

export default function Arch() {
  return <FlowDiagram dsl={dsl} style="sleek" />
}

Pass a graph object instead of DSL via config={...}. Full prop list in the API docs.

1. Install

zsh

2. Register the custom element

main.ts
// Register the <flow-diagram> custom element once
import 'flow-diagram/dist/wrappers/angular.js'

// In your AppModule add CUSTOM_ELEMENTS_SCHEMA
import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core'
@NgModule({ schemas: [CUSTOM_ELEMENTS_SCHEMA] }) class AppModule {}

3. Use it in a template

app.component.html
<flow-diagram
  [attr.dsl]="dsl"
  diagram-style="sleek"
  height="420px"
  (node-click)="onNode($event)">
</flow-diagram>

Uses the Web Components bridge — works with any Angular version that supports custom elements.

1. Drop the bundle in

index.html
<!-- Local copy after `npm install flow-diagram` -->
<script src="./node_modules/flow-diagram/dist/flow-diagram.standalone.js"></script>

<div id="arch" style="height:420px"></div>
<script>
  const dsl = `
nodes:
  - id: client, kind: actor,   label: Client
  - id: api,    kind: service, label: API
  - id: db,     kind: store,   label: Postgres

edges:
  - client -> api
  - api    -> db
  `
  // mount() gives you zoom / pan / fullscreen / download controls
  const vp = FlowDiagram.mount('#arch', {
    graph: FlowDiagram.parseDSL(dsl),
    styleName: 'sleek',
  })
  // vp.setStyle('city'); vp.update({ activeNodes: ['client'] })
</script>

Prefer the web-component form? <flow-diagram dsl="..." diagram-style="city" height="420px"></flow-diagram> — it wires up the same interactive viewport automatically.

1. Install

zsh
$ npm install flow-diagram

2. Render SVG server-side

build-diagrams.mjs
import { writeFileSync } from 'node:fs'
import { parseDSL, renderSVG } from 'flow-diagram'

const dsl = `
nodes:
  - id: client, kind: actor,   label: Client
  - id: api,    kind: service, label: API
  - id: db,     kind: store,   label: Postgres

edges:
  - client -> api, label: HTTPS
  - api    -> db,  label: query
`

const svg = renderSVG(parseDSL(dsl), { styleName: 'sleek' })
writeFileSync('docs/architecture.svg', svg)
console.log('wrote docs/architecture.svg')

Generate diagrams at build time for README files, static docs sites, PDF reports, and more.

Bundle exposes window.FlowDiagram — no build step required. Pin a specific version with flow-diagram@0.1.0. Listen for node-click / edge-click CustomEvents on the element.

The DSL in 10 seconds

Write it. Flow lays it out.

No x/y needed — ranks are computed from edges and the canvas sizes itself. Switch the style: directive to re-render the same graph in any of the five styles.

DSL reference →
hero.flow 
# A minimal, clean DSL — auto-layout handles the rest.
style: sleek

nodes:
  - id: client,  kind: actor,   label: Client
  - id: gateway, kind: gateway, label: "API Gateway"
  - id: orders,  kind: service, label: Orders
  - id: cache,   kind: cache,   label: Redis
  - id: db,      kind: store,   label: Postgres

edges:
  - client  -> gateway, label: "HTTPS"
  - gateway -> orders,  label: "POST /order"
  - orders ..> cache,   label: "read"
  - orders  -> db,      label: "write"
