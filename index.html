Diagrams-as-data · for docs, dashboards, decks

Draw systems the
way you talk about them.

A visual library for system, process, and architecture diagrams. Write a few lines of config; get a diagram you can render in four distinct aesthetics, animate step-by-step, and drop into docs, a Notion page, or a Docusaurus site.

Sleek · Sketch · Iso · Blueprint

Runtime

01 / player

Press play on your architecture.

The step-player narrates a system: highlight relevant nodes, animate the edges that carry traffic, pulse active services. Ship docs that read themselves.
[player — wires up below]
02 / styles

One diagram, four voices.

Swap style: "sleek" for "sketch" and the semantics stay; only the skin changes. Styles are pluggable — add your own by implementing the renderer contract.
[four-up style grid]
03 / nodes

A library of shapes, not drawings.

Every node is a typed primitive — service, store, queue, actor, boundary. The style layer decides how each type is drawn.
[node catalog]
04 / edges

Connections that move.

Solid, dashed, labeled, branching — with a flowing-packet animation that the player drives during step playback.
[edge catalog]
05 / config

Write a diagram like you write a list.

A tiny YAML-ish DSL for the 80% case, React components for the rest. Both compile to the same graph IR.
[DSL ↔ rendered diagram]
06 / playground

Try it. Right here.

Edit the DSL and watch every keystroke compile. Swap aesthetics. Pick a starter. Share a URL of exactly what you see.

07 / architecture

Tokens in, themes out.

Three-layer tokens (primitive → semantic → style-theme), a renderer contract, and a motion timeline. New style = new theme file + 6 renderer functions.
[token architecture]
